As it follows from the documentation, a special Datasource object needs to be passed to the *uiScroll directive. The basic ngx-ui-scroll example includes primitive callback-based Datasource.get method implementation with no additional settings. The use of the *uiScroll directive at the template layer is very similar to the use of *ngFor directive in the simplest case. In addition to Datasource, a special viewport container is specified at the template. It defines the scrollable area via height and overflow css properties.

Datasource.get log shows us that there are 4 requests of 5 items on the initial load (before scrolling). So we have 20 DOM elements (4 * 5) initially. The number of DOM elements is not constant during user scrolling, but it fluctuates around the initial value. Why do we get 5 items per each datasource.get call? It is so because of default. We may change it via settings object and bufferSize property.
